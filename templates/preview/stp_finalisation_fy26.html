STP Phase 2 finalisation · FY26

STP Phase 2 (mandatory 1-Jan-2022) replaced the PAYG payment summary with a year-to-date income statement in each employee’s myGov account. The finalisation declaration under TAA Sched 1 s.389-15 tells the ATO “these YTD figures are final” and flips each employee’s income statement to Tax ready. Due 14 July 2026 for arm’s-length employees, 30 September 2026 for closely-held payees (PCG 2018/D9). SAE Books reconciles every YTD bucket against the GL, validates Phase 2 disaggregation, surfaces RFBA + RESC, and signs the declaration once green across all 14 employees.

Market read

Xero / MYOB / Reckon
  • Xero Payroll — Phase 2 transition shipped 2022. Finalisation screen lists employees with YTD totals and a single Finalise All button; allowance disaggregation lives on the pay-item; RFBA/RESC entered manually; closely-held option toggled per employee.
  • MYOB Business — equivalent finalisation surface; PCG 2018/D9 closely-held option requires a separate workflow for company directors.
  • Reckon One — basic Phase 2 reporting; closely-held option C handled by tax-agent memo.
  • KeyPay (Employment Hero) — better leave-category granularity but RFBA still comes from a separate FBT register the user maintains in Excel.
SAE Books · native
  • YTD-to-GL traceability per employee: every $1 of W1 / 1A / RESC / RFBA reconciles to specific GL lines, not a wages-control total.
  • Phase 2 schema validation runs as pay events lodge; if a pay-code lands on an allowance with no Phase 2 code mapping (or a leave hour with no category), the pay run is blocked until classified.
  • Tax-treatment code re-derived per pay event from TFN dec + Withholding dec + country code; SAE warns if the derived code would have changed since the last pay run.
  • RFBA pulls live from the FBT workpaper (Fire 51), RESC from the salary-sacrifice register, ETP from the termination workpaper — no re-keying.
  • Closely-held option C runs as a quarterly estimator with 30-Sep true-up, wired into the same calendar as PAYG instalments and ATO Stapled Super lookups.
  • Finalisation declaration logs the s.389-25 authorisation chain; update events post-finalisation handle YTD corrections without re-running the full data set.
