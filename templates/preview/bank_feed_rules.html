How a feed line flows

  1. 1CDR pull — direct feed from CBA Open Banking (BSB+account → DataRecipient endpoint, signed JWT). Refresh window 24 h, max-age 25 h.
  2. 2Dedupe — by (date, amount, normalised description hash). MTD posts collapse to single row.
  3. 3Rule pass — first match by priority. If matched and conf ≥ 0.85, auto-post to draft journal.
  4. 4ML pass — for unmatched, predict CoA. If ≥ 0.85, post with ml=true; else queue for review.
  5. 5Reconciliation — match against open invoices/bills via reference, amount tolerance ±A$0.05, date window ±5 d.
  6. 6User confirm — every accept trains the model + suggests new rule after 3 matches.
