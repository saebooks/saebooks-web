Market read · why SAE Books exposes this surface

Xero / MYOB / Reckon · ATO data-matching
Xero (Practice Manager)
Pre-fill from ATO Tax Agent Portal at lodgement time.
Receives pre-fill after beneficiaries file — mismatch is detected post-hoc. No active scanning of bank/CRS/CHESS feeds against the GL.
MYOB (AE / AO)
Portal pre-fill + manual review checklist.
Tax-agent driven. No automatic TBR cross-match against trust resolution. AUSTRAC, MVR, lifestyle assets — all left to the agent to remember to check.
Reckon APS
APS Tax pre-fill + workflow.
Mid-tier agent suite. Data-matching detection is by exception only — no per-program registry, no risk-score panel.
SAE Books · this surface
Pre-match before lodgement; weekly intensity sweep.
31-program registry kept current with ATO gazette feed. Party-graph (/preview/parties) is one record per entity, so a discrepancy in one program (TBR · Saueesti) propagates into M Sauer's pre-lodgement file automatically.
